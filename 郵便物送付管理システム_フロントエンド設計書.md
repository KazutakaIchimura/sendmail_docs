## 2. アクセシビリティ方針

スタッフが利用者（障害者）の情報を**見やすく・ミスなく**操作できることを重視する。

| 方針 | 具体的な対応 |
|---|---|
| WCAG AA準拠 | コントラスト比4.5:1以上・フォーカス可視化 |
| コンポーネント品質 | デジタル庁デザインシステム（DADS）を使用（アクセシビリティスペシャリスト監修・WAI-ARIA準拠済み） |
| 操作の確認 | 送付済み更新など重要操作には確認モーダルを必ず挟む |
| 視覚的な状態表示 | 色だけに頼らずアイコン・テキストも併用（月遅れ・送付待ちなど） |
| エラー表示 | 具体的で分かりやすい言葉で表示 |

#### 表示ロジック

| 項目 | ロジック |
|---|---|
| 送付待ち件数 | `status='PENDING' AND send_month = 当月` |
| 送付済み件数 | `status='SENT' AND send_month = 当月` |
| 完了件数 | `status='DONE' AND send_month = 当月` |
| 月遅れアラート | `status='PENDING' AND send_month < 当月`（1件以上で表示） |
| 最近の履歴 | `status IN ('SENT','DONE') ORDER BY updated_at DESC LIMIT 5` |

| ルール | 内容 |
|---|---|
| 自己無効化禁止 | 自分自身のアカウントは無効化できない |

## 4. テスト環境設計（フロントエンド）

### 4.1 フロントエンド テスト

#### テストライブラリ

| ライブラリ | 用途 |
|---|---|
| Vitest | テストランナー（Viteと統合、高速） |
| React Testing Library | コンポーネントテスト |
| MSW（Mock Service Worker） | APIモック |
| Playwright | E2Eテスト（ブラウザ操作） |

#### テスト方針

| 対象 | テスト種別 | 優先度 |
|---|---|---|
| 送付先別一覧ビュー（月遅れ表示） | コンポーネントテスト | 高 |
| まとめて送付済みにするフロー | E2Eテスト | 高 |
| 送付物作成（重複チェック） | コンポーネントテスト | 高 |
| ダッシュボード表示 | コンポーネントテスト | 中 |
| 利用者・事業所CRUD | コンポーネントテスト | 中 |

#### ディレクトリ構成

```
frontend/src/
├── pages/
│   └── MailSendByOffice/
│       ├── index.tsx
│       └── index.test.tsx          ← コンポーネントテスト
├── __mocks__/
│   └── handlers.ts                 ← MSWのAPIモック定義
└── e2e/
    └── mailSendFlow.spec.ts        ← PlaywrightのE2Eテスト
```

#### コンポーネントテスト例

```typescript
// 月遅れの赤表示テスト
test('過去月の送付待ちは赤バッジで表示される', async () => {
  const overdueRecord = {
    id: 1, userName: '田中太郎',
    sendMonth: '2025-04', isOverdue: true, status: 'PENDING'
  };
  render(<MailSendRow record={overdueRecord} />);

  expect(screen.getByText('遅れ')).toHaveClass('text-red-600');
});
```

```
    └── Testcontainersが自動でPostgreSQLコンテナを起動
  フロントエンド：npm run test
    └── VitestがMSWでAPIをモック

【CI環境（GitHub Actions）】
  push / PRのたびに自動実行
    ├── バックエンドテスト（JUnit5 + Testcontainers）
    └── フロントエンドテスト（Vitest）
  E2Eテスト（Playwright）
    └── mainブランチへのマージ時のみ実行

【Heroku本番環境】
  テストは実行しない（CIで担保済み）
```

#### GitHub Actions 設定（概要）

> ※ GitHub Actions の設定ファイルはyaml形式が必須のため `.github/workflows/ci.yml` はyamlのまま使用する。

```yaml
# .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: '21' }
      - run: ./gradlew test

  frontend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci && npm run test
        working-directory: frontend
```

```tsx
    >
```

```
text-std-17B-7   → 見出し小（17px Bold）
text-std-16N-7   → 本文（16px Normal）
text-std-14N-7   → 補足テキスト（14px Normal）
text-std-14B-7   → 補足テキストBold（14px Bold）
```
