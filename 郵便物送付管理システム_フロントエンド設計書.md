## 2. アクセシビリティ方針

スタッフが利用者（障害者）の情報を**見やすく・ミスなく**操作できることを重視する。

| 方針 | 具体的な対応 |
|---|---|
| WCAG AA準拠 | コントラスト比4.5:1以上・フォーカス可視化 |
| コンポーネント品質 | デジタル庁デザインシステム（DADS）を使用（アクセシビリティスペシャリスト監修・WAI-ARIA準拠済み） |
| 操作の確認 | 送付済み更新など重要操作には確認モーダルを必ず挟む |
| 視覚的な状態表示 | 色だけに頼らずアイコン・テキストも併用（月遅れ・送付待ちなど） |
| エラー表示 | 具体的で分かりやすい言葉で表示 |

> **2026-06-21追加：認知アクセシビリティ方針**
> DADSが保証するWAI-ARIA準拠（スクリーンリーダー対応等）とは別軸で、知的障害・精神障害のあるスタッフも使うことを想定した「分かりやすさ」の改善を、アクセス頻度の高い画面から順に進めている。観点は以下の5点。
>
> 1. 一度に提示する情報・選択肢を減らす（ステップウィザード化、1画面の項目数を絞る）
> 2. エラーメッセージは平易な言葉で、原因と次の対処（再試行ボタン等）を明示する
> 3. 破壊的操作・状態変化（選択解除など）は無言で起こさず、何が起きたかを明示するフィードバックを出す
> 4. 取り消し可能な操作は確認モーダルを徹底し、件数や対象の具体的な一覧を見せる
> 5. 時間制限・自動タイムアウトは避ける
>
> 2026-06-22時点で、ダッシュボード・送付先別一覧・送付物作成・送付履歴の4画面に対応済み（詳細は各画面の節を参照）。

#### 表示ロジック

| 項目 | ロジック |
|---|---|
| 送付待ち件数 | `status='PENDING' AND send_month = 当月` |
| 送付済み件数 | `status='SENT' AND send_month = 当月` |
| 完了件数 | `status='DONE' AND send_month = 当月` |

| 最近の履歴 | `status IN ('SENT','DONE') ORDER BY updated_at DESC LIMIT 5` |

#### UXのポイント

- 送付先事業所は全件表示しチェックボックスで複数選択（紐付け絞り込みはしない。2026-06-21修正で反映済み）
- **2026-06-22追加**: 利用者選択ステップで対象者が多い場合に探しにくくなるため、氏名・ふりがなによる絞り込み入力を追加。0件時は「登録されている利用者がいません」とは区別し、「『〇〇』に一致する利用者が見つかりません」と表示する（絞り込みすぎたことが分かるようにする）
- 送付月はデフォルトで当月を選択済みにする
- 同一の利用者×事業所×種別×月が既に登録済みの場合は警告を表示（重複防止）
- 登録は「選択した事業所 × 送付種別」の組み合わせ単位で並行実行する（`Promise.allSettled`）。一部の組み合わせのみ失敗した場合（例: 一部だけ重複登録だった）は成功分を確定させたまま、確認画面に**失敗した組み合わせのみ**を再表示し「前回失敗した◯件のみを再送信します」と明示する。再送信時の確認画面は最初に選択した全件ではなく、残った組み合わせの事業所・送付種別のみを表示する（2026-06-21実装、2026-06-22に確認画面の表示不整合を修正）

```typescript
    .regex(/^(?=.*[a-zA-Z])(?=.*\d).+$/,
      '新しいパスワードは8文字以上で、英字（a〜z）と数字（0〜9）をまぜて設定してください'),
  confirmPassword: z.string(),
}).refine(d => d.newPassword === d.confirmPassword, {
  message: '上で入力した新しいパスワードと同じ内容を入力してください',
  path: ['confirmPassword'],
});
```

#### DADSのタイポグラフィクラス（よく使うもの）

DADSはTailwindのカスタムクラスでフォントサイズ・ウェイト・行間を指定します（`text-std-{サイズ}{B|N}-{行間}`の形式。末尾は固定の`-7`ではなく実際の行間値）。

```
text-std-32B-150  → 見出し大（32px Bold）
text-std-24B-150  → 見出し中（24px Bold）
text-std-17B-170  → 見出し小（17px Bold）
text-std-16N-170  → 本文（16px Normal）
text-std-14N-130  → 補足テキスト（14px Normal）
text-std-14B-130  → 補足テキストBold（14px Bold）
```

> 実際に使われている値は `frontend/src/pages/**` を `text-std-` で検索して確認すること（行間サフィックスはコンポーネントごとに異なる）。
